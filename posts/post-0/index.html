What better way to start a website than descibing how to setup a website. This is how I did it on my linux machine

In this post I'll be explaining how to host a website on Github and automate the deployment task using Github actions.

GitHub

A platform primarily to store and share code with version control history (git). It can be used to host static webpages.

Static webpages

These are websites that have a fixed content. The HTML, CSS, JS files needed to render these sites are already present and just served to the viewer on request, unlike *Dynamic webpages* that creates the pages tailored to the viewer when requested. Static pages load faster than dynamic pages.

Zola

It's a static site generator written in Rust. Writing HTML and CSS manually is cumbersome, so instead we'll write the content of the website in MarkDown format and let Zola make the required files from it based on a theme of our choice.

GitHub Actions

It's a kind of automation where we ask GitHub to do certain things (like running a script) when certain events happen (like update of a repo)

At the end the structure will be something ike this,

flowchart LR; md[MarkDown]-->|Zola|files[Static Files]; Theme/Template-->|Zola|files; files-->|Git|code[Website code repo in GitHub] code-->|GitHub Action|web[website repo] web-->|Git pages|website

Setting up GitHub

  1. I hope you already have a github account because that's where we'll be hosting the website.
  2. Make a code reposiitory in GitHub (say CodeRepo).This is wehre the whole content (markdowns, templates themes etc) used by Zola will be stored. You can make it a private repository if you want.
  3. Make another repository (name it <YourGithubName>.github.io for a simpler web domain name). This is where the static pages generated by Zola will be stored and then displayed as a website.
  4. Setup access to your GitHub from terminal so that you can push changes to GitHub. (Search online on how to do it)
  5. Clone your CodeRepo in a directory where you are going to keep the website files. Git should be installed.
git clone git@github.com:<YourGithubName>/CodeRepo.git

Create Zola website

  1. Install Zola.
  2. Initialise the CodeRepo directory as a web-directory.
zola init
  1. Make a Zola website in the directory (Search online on how to do it).
  2. Once the website is working locally make the static files
zola build
  1. Push the files to upstream/remote/GitHub
git push

Setup Deploy chains

CodeRepo configuration

  1. We'll make a GitHub Action that copies the public directory of CodeRepo to the WebRepo i.e.<YourGithubName>.github.io.
  2. Go to GitHub, and navigate to the settings/Actions/General of CodeRepo and set
    • Actions permission to "Allow all actions and reusable workflows"
    • Workflow permissions to "Read and write permissions"
  3. Add an api token to secrets
  4. Now, go to Actions tab of CodeRepo and click on "Setup a new workflow yourself". Paste the following and edit the destination-github-username, repository-name and user-email.
name: Push public to website repo

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v2
      - name: Pushes to another repository
        uses: cpina/github-action-push-to-another-repository@main
        env:
          API_TOKEN_GITHUB: ${{ secrets.API_TOKEN_GITHUB }}
        with:
          source-directory: 'public'
          destination-github-username: 'pritipriya-dasbehera'
          destination-repository-name: 'pritipriya-dasbehera.github.io'
          user-email: pritipriya.dasbehera@gmail.com
          target-branch: main
  1. Save the Action. If everything goes well, GitHub should copy the public directory to WebRepo. Else, run the Action manually.

WebRepo configuration

  1. Now, we need to host the website from WebRepo. Navigate to settings/pages of WebRepo.
  2. Select Branch (None will be selected previously) to main and folder to /(root), then save.
  3. Click on Source and select "Deploy from a branch"
  4. Soon you should get the website link pop up.

Commit and check

As a final check, make some change locally on your computer and push them to GitHub, the website should be updated in a few minutes automatically.
